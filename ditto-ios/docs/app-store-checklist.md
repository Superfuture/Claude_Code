# App Store Submission Checklist

Pre-flight everything below before tapping "Submit for Review."

## Apple Developer Program

- [ ] Enrolled at developer.apple.com ($99/yr)
- [ ] Two-factor auth enabled on your Apple ID
- [ ] Agreed to latest Paid Applications agreement (required for IAP)
- [ ] Tax & banking info filled in App Store Connect → Agreements

## Identifiers & signing

- [ ] App ID created: `com.yourdomain.ditto`
- [ ] App ID created for extension: `com.yourdomain.ditto.MessagesExtension`
- [ ] App Group registered: `group.com.yourdomain.ditto`
- [ ] Both targets enable App Groups capability with the same group ID
- [ ] Both targets signed with the same Team
- [ ] Distribution provisioning profile created (or use Xcode Automatic)

## In-App Purchase

- [ ] Subscription Group created in App Store Connect
- [ ] Auto-Renewable Subscription `ditto.pro.monthly` ($4.99/mo, 7-day intro free trial)
- [ ] Auto-Renewable Subscription `ditto.pro.annual` ($29.99/yr)
- [ ] Both products in "Ready to Submit" state
- [ ] Subscription privacy & terms URLs set
- [ ] StoreKit2 (or RevenueCat) wired in app — Pro entitlement updates AppGroupStore.isPro

## App Store Connect listing

- [ ] App name: `Ditto - Smart Reply Suggestions` (under 30 chars)
- [ ] Subtitle (under 30 chars): `Three replies, inside iMessage`
- [ ] Primary category: Social Networking (or Productivity)
- [ ] Keywords (100 chars total): `imessage,replies,ai,smart,suggestions,texting,messages,reply,chat,quick`
- [ ] Description (4000 char max) — explain what Ditto does, the privacy posture, the iMessage extension setup
- [ ] What's New: "Initial release"
- [ ] Support URL: must resolve and have basic support content
- [ ] Marketing URL: optional, recommend pointing to ditto.app
- [ ] Promotional text (170 char max): optional
- [ ] Copyright: `© 2026 Your Name`

## Privacy

- [ ] Privacy Policy URL — hosted publicly, must cover: what data is sent (message content), where it goes (Anthropic via our proxy), retention (none), no training, contact info
- [ ] App Privacy section in App Store Connect:
  - Data Types Collected: User Content (Messages)
  - Linked to user: No
  - Used for tracking: No
  - Purpose: App Functionality

## Apple App Review prep

- [ ] Demo account: not needed (no login)
- [ ] Notes for reviewer:
  > Ditto is an iMessage app extension. To test, open Messages on the simulator/device, tap any conversation, tap the iMessage Apps icon (next to the input), and pick Ditto. Tap a tone preset to generate three reply suggestions. Tapping a suggestion inserts it into the iMessage draft field. Subscription is optional — five free generations per day on the free tier.
- [ ] **AI disclosure** in description: state clearly that suggestions are AI-generated by an LLM
- [ ] **Content rating questionnaire** completed — likely 12+ for the Flirty tone preset

## Assets (uploaded to App Store Connect)

- [ ] App icon 1024×1024 PNG (no transparency, no rounded corners — Apple rounds it)
- [ ] iPhone screenshots:
  - 6.7" (iPhone 15 Pro Max): 1290×2796 — **required**
  - 6.5" (iPhone 11 Pro Max): 1242×2688 — required as fallback
- [ ] Recommended: 5 screenshots showing onboarding, the iMessage extension in action (multiple tones), and the paywall
- [ ] iMessage extension icon set (in Xcode asset catalog) — separate from main app icon

## TestFlight

- [ ] Archive built via Xcode (Product → Archive)
- [ ] Uploaded to App Store Connect via Organizer
- [ ] Internal testing group set up, invite yourself first
- [ ] At least 1 round of testing on a real device — extension actually appears in Messages
- [ ] Subscription purchase works in sandbox (test Apple ID)

## Submit

- [ ] Build attached to the version in App Store Connect
- [ ] Export Compliance answered (no encryption beyond Apple's standard frameworks)
- [ ] Submitted for review

## After submission

- Expected review time: 24–72 hours, sometimes longer for AI/extension apps
- Common rejection reasons to avoid in advance:
  - Inadequate AI disclosure in description
  - Privacy policy URL 404
  - Demo flow not clear for reviewer (always include step-by-step in reviewer notes)
  - Subscription products not ready in App Store Connect
- If rejected: Apple sends a Resolution Center message. Reply, fix, resubmit.
